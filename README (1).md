# 🇺🇦 Ukraine air raid alert — time series analysis

A pet project that combines a **Python data pipeline** and a **static HTML dashboard**
to visualise air raid alert patterns across all 25 regions of Ukraine from
February 24, 2022 to June 20, 2026.

> Built in two days as a beginner-friendly mini-project. Data is public and sourced
> from [alerts.in.ua](https://alerts.in.ua).

---

## Live demo

Open `ukraine_alerts_v4.html` in a browser — or host it on GitHub Pages (see below).
The dashboard works offline with built-in fallback data and upgrades automatically
when `outputs/dashboard_data.json` is present.

## What the project does

### Python pipeline (`main.py`)

Runs five steps in sequence when you type `python main.py`:

| Step | File | What it does |
|------|------|-------------|
| 1 | `load_and_clean.py` | Reads the CSV, parses dates, removes broken rows, adds time columns |
| 2 | `analyze.py` | Calculates totals, averages, top-10 rankings, monthly trends, longest alerts |
| 3 | `export_json.py` | Packages all results into `outputs/dashboard_data.json` |
| 4 | `make_charts.py` | Generates 6 publication-ready PNG charts |
| 5 | `main.py` (summary) | Prints a readable report to the terminal |

### HTML dashboard (`ukraine_alerts_v4.html`)

A fully self-contained single-file website with:

- **7 pages** — dashboard, timeline, map, compare, rankings, region detail, forecast, methodology
- **Choropleth map** of all 25 Ukrainian oblasts coloured by alert intensity
- **Photo of each regional city** embedded in the sidebar and region pages
- **Forecast model** — linear trend + seasonal adjustment, selectable horizon
- **Export buttons** — download CSV, print to PDF
- **JSON loader** — on startup the dashboard fetches `outputs/dashboard_data.json`
  (generated by Python) and populates all charts with real data
- **Offline fallback** — if the JSON is missing the dashboard still works with
  built-in data (useful when opening directly from disk)

```bash

```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

## Using your own data

Replace `data/alerts_sample.csv` with a real export from
[alerts.in.ua](https://alerts.in.ua) or the
[Vadimkin dataset](https://github.com/Vadimkin/ukrainian-air-raid-sirens-dataset).

Your CSV must contain at least these columns:

| Column | Example | Required |
|--------|---------|----------|
| `region` | `Kharkiv region` | ✅ |
| `start_time` | `2022-10-10 08:00:00` | ✅ |
| `end_time` | `2022-10-10 10:00:00` | optional |
| `duration_minutes` | `120` | optional |

Then run `python main.py` again — the dashboard updates automatically.

## Deploying to GitHub Pages

GitHub Pages hosts your dashboard for free at a public URL.

### Step 1 — Create a repository

Go to [github.com/new](https://github.com/new) and create a new public repository
named `ukraine-air-alerts`.

### Step 2 — Push the project

```bash
git init
git add .
git commit -m "Initial commit — Ukraine air raid alert dashboard"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/ukraine-air-alerts.git
git push -u origin main
```

### Step 3 — Enable GitHub Pages

1. Open your repository on GitHub
2. Go to **Settings → Pages**
3. Under **Source**, select `Deploy from a branch`
4. Select **branch: main**, folder: **/ (root)**
5. Click **Save**

Your dashboard will be live in about one minute at:

```
https://YOUR_USERNAME.github.io/ukraine-air-alerts/ukraine_alerts_v4.html
```

### Step 4 — Keep data fresh

Whenever you have new alert data:

```bash
# 1. Update your CSV in data/
# 2. Regenerate the JSON and charts
python main.py

# 3. Commit and push
git add outputs/dashboard_data.json outputs/charts/
git commit -m "Refresh dashboard data — $(date +%Y-%m-%d)"
git push
```

GitHub Pages redeploys automatically within seconds.

## How the dashboard loads data

When you open the dashboard in a browser it runs this sequence:

```
1. Show loading spinner
2. fetch("outputs/dashboard_data.json")
      ↓ success                  ↓ file not found / offline
   populate charts           use built-in fallback data
      ↓
3. Build all 7 pages
4. Hide spinner — dashboard is ready
```

This means the HTML file **always works** — with fresh Python-generated data when
available, or with built-in data when opened directly from disk or shared by email.

## Notes for beginners

- Each `src/` file does **one thing only** — easy to read, test, and modify.
- All analysis functions **return** their results rather than printing them — you can
  import and reuse them in a Jupyter notebook without any changes.
- `main.py` is the only file you normally need to run.
- The HTML dashboard is a **single file** — you can share it by email or USB and it
  opens in any browser even with no internet connection.
- The forecast is a simple statistical model, not a military or intelligence prediction.
  Do not use it for safety planning or official purposes.
